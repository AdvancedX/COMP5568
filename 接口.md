# 接口对齐规范（初稿）

> 目标：支持「稳定币 + wBTC 抵押」的借贷协议，前后端并行开发时，**以本文件为唯一接口对齐来源**。  
> 包含：合约接口（函数 + 事件）和关键数据结构。

---

## 1. 合约与组件列表

- `WBTCToken`（测试用 ERC20，抵押资产）
- `Stablecoin`（协议内部稳定币，ERC20）
- `LendingPool`（核心借贷逻辑）
- `PriceOracle`（��格与随机波动）

前端只直接与 **`LendingPool` / `WBTCToken` / `Stablecoin` / `PriceOracle`** 交互。

---

## 2. 全局约定

- **链环境**：本地测试链（Hardhat / Foundry / Anvil 等，待定）。
- **精度约定（建议，待最终确认并统一）**：
  - `wBTC.decimals = 8` 或 `18`（需要最终拍板）
  - `Stablecoin.decimals = 18`
  - 利率、比例（如抵押率、健康度）：`1e18` 精度（如 0.75 表示为 `0.75e18`）
  - 价格：`getWbtcPrice()` 返回单位为 `1e18`（1 wBTC 的价格，以 1 稳定币 = 1 USD 为基准）

前端在显示数值时，必须根据以上精度进行换算。

---

## 3. ERC20 通用接口（wBTC & Stablecoin）

前端将与以下标准 ERC20 接口交互（两个代币均一致）：

```solidity
function name() external view returns (string);
function symbol() external view returns (string);
function decimals() external view returns (uint8);

function balanceOf(address account) external view returns (uint256);
function allowance(address owner, address spender) external view returns (uint256);
function approve(address spender, uint256 amount) external returns (bool);
```

### 前端调用约定

- 读余额：
  - `wbtc.balanceOf(user)`
  - `stable.balanceOf(user)`
- 授权：
  - 抵押前：`wbtc.approve(lendingPool, amount)`
  - 还款前：`stable.approve(lendingPool, amount)`

## 4. LendingPool 接口

### 4.1 数据结构

```solidity
struct UserAccount {
    uint256 collateralWbtc;   // 抵押的 wBTC 数量
    uint256 debtStable;       // 借出的稳定币数量
}
```

### 4.2 只读查询（view）

```solidity
/// @notice 返回用户的持仓信息
function getUserAccount(address user) external view returns (UserAccount memory);

/// @notice 抵押因子（决定最大可借额度），如 0.75e18
function getCollateralFactor() external view returns (uint256);

/// @notice 清算阈值，如 0.80e18
function getLiquidationThreshold() external view returns (uint256);

/// @notice 平台预留利息比例，如 0.10e18
function getReserveFactor() external view returns (uint256);

/// @notice 当前借款利率（按区块计）
function getBorrowRatePerBlock() external view returns (uint256);

/// @notice 当前存款利率（按区块计）
function getSupplyRatePerBlock() external view returns (uint256);

/// @notice 资产合约地址
function getWbtcAddress() external view returns (address);
function getStablecoinAddress() external view returns (address);
```

#### 风险 & 估算相关

```solidity
/// @notice 返回用户健康度（>1 安全，=1 清算边界，<1 应被清算）
/// 精度：1e18
function getHealthFactor(address user) external view returns (uint256);

/// @notice 用户抵押资产价值（以稳定币/美元计价）
/// 单位与 Stablecoin 一致（18 decimals）
function getCollateralValue(address user) external view returns (uint256);

/// @notice 用户债务价值（以稳定币/美元计价）
function getDebtValue(address user) external view returns (uint256);

/// @notice 计算用户当下最大可借的稳定币数量
function getMaxBorrowable(address user) external view returns (uint256);
```

### 4.3 状态变更接口（核心四个逻辑）

#### 4.3.1 Deposit（供应 / 抵押）

```solidity
/// @notice 抵押 wBTC 到协议中
/// @param amount 要抵押的 wBTC 数量（使用 wBTC 的 decimals）
function deposit(uint256 amount) external;
```

**前端调用流程：**

1. 读取 `wbtc.allowance(user, lendingPool)`.
2. 不足时，先调用 `wbtc.approve(lendingPool, amount)`.
3. 调用 `lendingPool.deposit(amount)`.
4. 成功后刷新：
   - `getUserAccount(user)`
   - `getHealthFactor(user)`
   - `getCollateralValue(user)`

**事件：**

```solidity
event Deposited(address indexed user, uint256 amount);
```

---

#### 4.3.2 Withdraw（提取抵押）

```solidity
/// @notice 提取部分或全部抵押的 wBTC
/// @param amount 要提取的 wBTC 数量
function withdraw(uint256 amount) external;
```

合约会确保：提取后健康度不低于阈值。

**前端调用流程：**

1. 读取当前抵押：`getUserAccount(user)`.
2. 根据前端输入 `amount`，调用 `withdraw(amount)`.
3. 刷新：
   - `getUserAccount(user)`
   - `getHealthFactor(user)`
   - `getCollateralValue(user)`

**事件：**

```solidity
event Withdrawn(address indexed user, uint256 amount);
```

---

#### 4.3.3 Borrow（借稳定币）

```solidity
/// @notice 基于抵押 wBTC 借出稳定币
/// @param amount 要借出的稳定币数量
function borrow(uint256 amount) external;
```

**前端调用流程：**

1. 查询 `getMaxBorrowable(user)`，在 UI 上展示最大可借额度。
2. 用户输入借款 `amount`（不可超过最大值���。
3. 调用 `borrow(amount)`.
4. 刷新：
   - `getUserAccount(user)`
   - `stable.balanceOf(user)`
   - `getHealthFactor(user)`
   - `getDebtValue(user)`

**事件：**

```solidity
event Borrowed(address indexed user, uint256 amount);
```

---

#### 4.3.4 Repay（还款）

```solidity
/// @notice 还稳定币
/// @param amount 要还的稳定��数量（应允许大于当前债务时视为全额还）
function repay(uint256 amount) external;
```

**前端调用流程：**

1. 读取 `stable.allowance(user, lendingPool)`.
2. 不足时先调用 `stable.approve(lendingPool, amount)`.
3. 调用 `repay(amount)`.
4. 刷新：
   - `getUserAccount(user)`
   - `stable.balanceOf(user)`
   - `getHealthFactor(user)`
   - `getDebtValue(user)`

**事件：**

```solidity
event Repaid(address indexed user, uint256 amount);
```

---

#### 4.3.5 Liquidate（清算）

```solidity
/// @notice 对健康度 < 1 的账户发起清算
/// @dev 清算人偿还稳定币债务，按清算奖励比例获得被清算人的 wBTC 抵押
/// @param user 被清算用户
/// @param repayAmount 希望偿还的稳定币数量（实际偿还会受债务和抵押价值上限限制）
function liquidate(address user, uint256 repayAmount) external;

/// @notice 清算奖励比例（1e18 精度），当前实现为 0.05e18（5%）
function getLiquidationBonus() external view returns (uint256);
```

**前端调用流程：**

1. 查询 `getHealthFactor(user)`，仅当 < `1e18` 时允许清算。
2. 清算人先调用 `stable.approve(lendingPool, repayAmount)`。
3. 调用 `liquidate(user, repayAmount)`。
4. 刷新：
   - `getUserAccount(user)`
   - `getHealthFactor(user)`
   - `wbtc.balanceOf(liquidator)`

**事件：**

```solidity
event Liquidated(
  address indexed liquidator,
  address indexed user,
  uint256 repaidStable,
  uint256 seizedWbtc
);
```

## 5. PriceOracle 接口（价格 + 随机变化）

为简化，仅提供 wBTC 价格：

```solidity
/// @notice 获取当前 wBTC 价格
/// 建议：返回 1 wBTC 对 Stablecoin（USD）的价格，精度 1e18
function getWbtcPrice() external view returns (uint256);
```

可选：用于开发环境随机价格模拟的接口（仅 owner 或脚本调用）：

```solidity
/// @notice 更新价格（例如内部用伪随机值上下浮动）
/// 生产环境可移除或改为安全预言机
function updatePrice() external;
```

**事件（可选）：**

```solidity
event PriceUpdated(uint256 newPrice);
```

前端可选择：
- 轮询 `getWbtcPrice()`；
- 或监听 `PriceUpdated`，自动刷新价格、健康度。
